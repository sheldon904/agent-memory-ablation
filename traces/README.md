# Exported recall traces

Every `recall()` call in the evaluation is instrumented as an OpenTelemetry-style
**span** and exported here, one subdirectory per arm. This ships a
standard-tooling telemetry artifact in the repository rather than a bespoke log
format, closing, as a side effect, the "we only speak our own telemetry" gap
the companion production study flagged about itself.

Regenerated by `make eval` (or `python -m harness.run_eval`).

## Layout

```
traces/
  rag/     spans.jsonl   otlp.trace.json
  graph/   spans.jsonl   otlp.trace.json
  hybrid/  spans.jsonl   otlp.trace.json
```

* **`spans.jsonl`**, one span per line, OTLP JSON encoding. 200 spans per arm
  (one per query).
* **`otlp.trace.json`**, the same spans wrapped in a single OTLP
  `resourceSpans` document, importable by any OTLP-aware tool.

## Span schema

Each span is `name = "memory.recall"` with these attributes:

| Attribute | Meaning |
|---|---|
| `memory.arm` | `rag` \| `graph` \| `hybrid` |
| `query.id` | e.g. `KI-001`, `MH-014`, `DX-003` |
| `query.type` | `known_item` \| `multi_hop` \| `distractor` |
| `recall.latency_ms` | measured wall-clock latency of the call |
| `recall.refused` | whether the arm reported "not in memory" |
| `recall.n_results` | number of facts returned |
| `recall.top_score` | score of the top result |
| `recall.top_fact` | fact id of the top result |
| `channel.*` | per-channel diagnostics (e.g. `channel.graph_seeds`, `channel.vector_hits`, `channel.bm25_hits`) |

`traceId` / `spanId` are derived deterministically from `(arm, query_id)` so span
identity is stable across runs; the span *duration* reflects the actual measured
latency, so re-running updates timings but not identities.

## Optional: Langfuse

If `langfuse` is installed and `LANGFUSE_PUBLIC_KEY` / `LANGFUSE_SECRET_KEY` are
set, each recall is *also* logged to Langfuse (`harness/tracing.py`,
`LangfuseExporter`). Absent creds or package, that sink is a silent no-op, the
OTLP export above is always produced and never depends on a running Langfuse
instance.
